# Bootstrapping Voilà

Voilà compiles itself. This file explains what that means here, how to rebuild
the toolchain from nothing, and how the claim is checked.

## The chain

```
   bootstrap/voilac.c          runtime/src/*.c
   (generated C, checked in)   (libvoila, hand-written C)
            │                          │
            └────────────┬─────────────┘
                         │  cc
                         ▼
                   voilac-seed  ──compiles voilac/*.voi──▶  voilac-1
                                                              │
                                     compiles voilac/*.voi ───┘
                                                              ▼
                                                          voilac-2

   FIXPOINT:  C(voilac-1)  ==  C(voilac-2)
```

`./build.sh` runs exactly this and fails loudly if the fixpoint does not hold.

The only thing you need is a C11 compiler. There is no Go, no prior Voilà
binary, no make, no CMake, no LLVM.

## What each piece is

| Piece | Language | Role |
|---|---|---|
| `voilac/` | **Voilà** | the compiler: `lex` → `par` → `load` → `chk` → `low` → `cg` |
| `runtime/src/` | C | libvoila: values, refcounts, strings, collections, decimals, exceptions, defers, tasks/channels/select, the std library |
| `bootstrap/voilac.c` | generated C | the seed — the C the compiler emits **for itself** |
| `voilac/rt/blob.voi` | generated Voilà | libvoila's source, carried *inside* the compiler so a produced binary needs nothing on disk |

`bootstrap/voilac.c` is not hand-maintained. It is regenerated by
`./build.sh reseed`, and `build.sh` warns when it has drifted from what the
compiler now emits.

## Why a seed is checked in

Self-hosting has a chicken-and-egg problem: to build the compiler you need the
compiler. Every self-hosted language answers it the same way — by shipping
*something* a foreign toolchain can consume. Voilà ships the compiler's own C
output. It is large (~36 000 lines) and unreadable, and that is fine: it is not
source, it is a fossil. The real source is `voilac/*.voi`, and the fossil is
verified against it on every build.

## Verifying the claim yourself

```sh
./build.sh          # seed → voilac-1 → voilac-2, fixpoint asserted
./selftest.bash     # the self-hosted toolchain reproduces every golden
```

`build.sh` asserts `C(voilac-1) == C(voilac-2)`: the compiler, compiled by
itself, emits byte-identical code for its own source. That is the classic
self-hosting fixpoint, and it is the strongest cheap evidence that the compiler
is not quietly mistranslating itself.

The binaries differ in ~49 bytes — the linker's UUID and the signature hash
computed over it. The C, which is what the compiler is responsible for, is
identical.

## How it was built (the historical part)

Stage 0 was a compiler written in **Go**: a tree-walking interpreter plus a C
backend. It served two purposes and then was deleted.

1. It **ran** the Voilà compiler while the Voilà compiler could not yet run.
2. It was the **oracle**. Every stage of the self-hosted compiler was diffed
   against it, byte for byte, over every `.voi` file in the repository: token
   streams, parse trees, flattened programs, IR listings, and the emitted C.
   `difftest.bash` did that, and it passed on all 76 sources — including the
   compiler's own.

The interpreter's semantics survive its deletion, frozen in the checked-in
goldens (`tests/conformance/*.out`, `samples/golden/*`). Those files were
produced by the Go tree-walker; the self-hosted toolchain is still held to them
by `selftest.bash`. The oracle is gone, but its verdicts are permanent.

### What the fixpoint caught that stage-by-stage diffing did not

Diffing each stage proves the two compilers *agree*. It does not prove either is
*right*, and it cannot see a bug that both engines are blind to. Compiling the
compiler with itself found two:

- **Register aliasing.** The lowerer bound a new name directly to the register
  its initialiser produced. That is correct for every expression that allocates
  one — but a bare identifier hands back the register of an *existing* binding,
  so `var a = b` made `a` and `b` the same slot, and `a = 1` silently changed
  `b`. Only the compiled engine was affected; the interpreter always copied. The
  compiler's own `lower_switch` does exactly `var defaultL = end`, so it
  miscompiled itself into a duplicate C label. Fixed by `freshen()`, which MOVEs
  into a fresh register; `tests/conformance/12_binding.voi` pins it.
- **`int(s, 0)`.** Go's `ParseInt` understands `0o`, `0b` and `_`; C's `strtoll`
  does not. The compiler parses integer literals with it, so `0o755` in any
  program compiled by a *natively built* compiler became a "bad integer literal".

Both are the same species: a divergence between the two engines that only a
program as large and as self-referential as the compiler was ever going to
provoke. That is what bootstrapping buys you.

## Deviations

- **Tasks are OS threads** (pthreads), not 4 KB green threads. The semantics —
  groups, cancellation, join, channels, `select` — are correct; the cost model
  is not. A ucontext scheduler is a later optimisation.
- **`std/regex` and `std/http` are not available in native builds.** They have no
  C implementation yet; the backend refuses rather than emitting something that
  would behave differently.
- **Values are boxed.** Performance is interpreter-class-plus, not C-class, until
  an unboxing pass lands. Correctness and self-hosting came first.
